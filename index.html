Everything locked until they log in

One password gates the whole server. Here's what it covers.

🔒

Total lockdown

Unauthenticated players can't move, break or place blocks, touch the inventory, chat, or take and deal damage.

⛔

Even OPs are gated

Command blocking runs before permissions resolve, so a prior-OP account can't use /op, /gamemode or anything else until it authenticates.

🛡️

Per-IP temporary ban

Failed attempts are counted per IP. Too many and that IP is blocked for a configurable time — stopping brute force from a single connection.

🌑

Blindness until login

A continuous Blindness effect hides the world from anyone who hasn't entered the password yet. It clears the instant they authenticate.

📡

Server-side only

Players connect with a normal vanilla or Fabric client. There is nothing to install on their side.

🌐

Multilingual

Ships with English and Spanish. Add your own language with a single JSON file — no code, no rebuild.

Up and running in five steps

On an existing Fabric 1.20.1 server. Takes about five minutes.

  1. 1

    Have Fabric + Fabric API ready

    Make sure your 1.20.1 server runs Fabric Loader and has Fabric API in its mods folder.

  2. 2

    Drop in the jar

    Copy the downloaded jar into the server's mods folder, next to Fabric API.

    mods/
├── fabric-api-0.92.2+1.20.1.jar
└── simpleserverpassword-1.0.0.jar
  3. 3

    Start once to generate config

    Boot the server. The mod creates its config and language files automatically, then stop the server.

    config/simpleserverpassword.json
config/simpleserverpassword/lang/{en,es}.json
  4. 4

    Set your password

    Open the config and change the password field. It's plain text, so don't reuse an important password.

    {
  "password": "my-secret-key",
  "maxAttempts": 3,
  "banDurationMinutes": 15,
  "language": "en"
}
  5. 5

    Restart — you're protected

    Start the server again. Players now have to run /password <key> before they can do anything.

FAQ

Do players need to install the mod too?

No. It's server-side only. Players join with a normal vanilla or Fabric client — there's nothing for them to install.

An OP can't run commands. Is that a bug?

No, it's intentional. Command blocking happens before permissions are resolved, so nobody — not even an OP — can run commands until they authenticate. Once they log in, all permissions come back.

I forgot the password or got banned. What now?

The password is plain text in the config file, so you can just read it. Bans live in memory, so restarting the server clears every ban and attempt counter instantly.

Do sessions persist between reconnects?

No, by design. Every new connection — including a reconnect — has to authenticate again. Nothing is stored between sessions.

Is the password hashed?

No. It's stored in plain text on purpose, for the simplicity of a private server. Don't reuse a password that matters elsewhere.

Why count failed attempts per IP instead of per account?

To stop an attacker cycling through several compromised accounts from the same connection. The per-IP counter shuts the whole connection down after too many tries.
